Overview of the Reference Application

The user has the ability to create / view / delete / update projects which contain references. The user can also manually add / view / delete references to and from a project. If the user does not specify the title of a reference, the WebAPI interface will extract the target page's title meta property and use that.

In addition the user can also install the bookmarklet below by dragging it into their favorites in their browser. Then, when on a page on any other site, if they click this button a popup will appear allowing the user to select a project and enter additional information about the reference to add to the project

+project reference - install add project reference bookmarklet

Understanding the Reference Application Makeup

This is a single page application (SPA) that is launched from a ASP.NET MVC application hosted in Microsoft Azure Web Site. An application has also been created in Azure Active Directory and granted permissions within SharePoint Online within Office 365. In order to launch the SPA, the user must authenticate using a valid Organization ID (OrgID) from their Office 365 subscription. The Azure Active Directory that the OrgID belongs to is in the same Azure subscription that contains the ASP.NET MVC application running in the Azure Web Site. All communication with Office 365 is handled by the ASP.NET WebAPI interface within the ASP.NET MVC application.

When a user logs in, the ASP.NET application first ensures the necessary lists reside in the specified SharePoint Online site. If they do not exist, they are created at that time... this is what is happening when the app says it is "initializing".
